Токен должен выдаваться user service, он же должен проводить валидацию
Храниться токен будет на стороне бота - и это единственная информация, которая должна быть доступна боту о сервисе авторизации, далее он будет передавать заголовок с токеном при обращении к сервисам, там в user service он будет проверять не закончился ли он и при необходимости получать новый вместе с ответом от сервиса
